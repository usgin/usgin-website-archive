USGIN Specifications

USGIN specifications are the standards, profiles, protocols, and hardware and software applications employed by USGIN to address the challenges associated with sharing large amounts of diverse data stored in geographically disparate locations. By using the specifications listed below, technical users can share their data in such a way that the data they publish is interoperable. For more information, see the USGIN specifications entry on the USGIN FAQ page.

USGIN specifications include and the support the following:

Category: Applications
Specification Description Uses

Amazon Web Services

Computing resources available for rent. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/amazon-web-services] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Several servers that host USGIN websites or provide USGIN services are provided by Amazon Web Services.

Apache Tomcat

Free-and-open-source web server software. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/apache-tomcat] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Used to run various software applications, such as GeoServer and GeoPortal, which provide Open Geospatial Consortium (OGC)-compliant web services

Django

Open-source web application framework written in the Python programming language. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/django] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Provides a number of USGIN sites and web services and custom applications, including the USGIN URI redirection engine

Drupal

Modular, extensible, actively maintained free-and-open-source web site content management framework; capable of running on any computing platform that supports PHP-capable web server software and database software. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/drupal] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Several websites related to the USGIN Initiative were built using the Drupal CMF, including:

  • The AASG Geothermal Data [Dead Link, http://www.stategeothermaldata.org/] website
  • The NDGS Portal [Dead Link, http://geothermaldata.org/]
  • The USGIN website

ESRI ArcGIS Desktop, ArcGIS Explorer, ArcGIS Server, and Geoportal

Proprietary geographic information system software and web service-compatible server software

Though USGIN specifications support open-source GIS software applications, USGIN users can, and often do, deploy their data as web services using ESRI software.

GeoServer

Free-and-open-source web service-compatible server software. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/geoserver] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Goeserver is a useful alternative for technical users with data to share who are disinclined to acquire a license for proprietary web service-compatible web server software.

As an example: Southern Methodist University, a contributor to theÂ  National Geothermal Data System [Dead Link, http://geothermaldata.org/], uses GeoServer to host web services; a Southern Methodist University GeoServer REST page is available here.

PostGIS

Adds support for geographic objects to PostgreSQL. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/postgresql-and-postgis] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

PostGIS is used by any number of databases that power Open Geospatial Consortium (OGC) web services.

PostgreSQL

Free-and-open-source object-relational database system with over 15 years of active development. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/postgresql-and-postgis] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

PostgreSQL is used by any number of databases that power Open Geospatial Consortium (OGC) web services.

Python

A programming language. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/applications/python] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Within the context of USGIN, Python is typically used for Django sites and converting Excel spreadsheets into ISO metadata (the application responsible for which can be found at http://github.com/usgin/csvtometadata).
Category: Web Service Standards
Specification Description Uses

ESRI Map Service

The standards governing web services deployed using ESRI software. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/esri-map-service] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

ESRI services can be produced by simply "publishing" an ESRI ArcMap document (.mxd) to an active ArcGIS Server

OGC Catalog Service for the Web (CSW 2.0.2)

The OpenGIS Catalogue Services Interface Standard (CAT) supports the ability to publish and search collections of descriptive information (metadata) about geospatial data, services, and related resources; CSW 2.0.2 is a CSW standard. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/ogc-catalog-service-web-csw-202] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Users typically deploy catalog services to list and manage queries regarding a large number of different web-accessible resources. Catalog services deployed by USGIN users in accordance with the OGC CSW 2.0.2 standard will demonstrate maximum compatibility with other OGC web services and a degree of compatibility with USGIN web services
Category: Metadata Standards
Specification Description Uses

FGDC Content Standard for Digital Geospatial Metadata

Provides a common set of terms and definitions for the documentation of digital geospatial data. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/fgdc-content-standard-digital-geospatial-metadata-csdgm] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Usage not recommended for USGIN-compliant materials.

ISO 19115

Standard for geospatial metadata; developed by the International Standards Organization (ISO). For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/iso-19115] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Governs metadata records describing web services.

ISO 19119

Standard for metadata describing geospatial web services. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/iso-19119] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Governs metadata records describing web services.

ISO 19139

Implementation of ISO 19115 and ISO 19119 standards. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/iso-19139] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

All USGIN-compliant metadata conforms to this standard.
Category: Schema Standards
Specification Description Uses

Geography Markup Language (GML)

An XML grammar for expressing geographical features; developed by OpenGIS. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/geography-markup-language] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Used to describe the location of features in any Open Geospatial Consortium (OGC) data services.

GeoSciML [Dead Link, http://www.cgi-iugs.org/tech_collaboration/geosciml.html]

GeoSciML is a standards-based XML data format for application-neutral encoding of geoscience data and related spatial data. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/geosciml-xml-markup-language] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

GeoSciML functions as an interchange format for USGIN users. Though data encoded by USGIN users in accordance with GeoSciML is interoperable and can be exchanged between application, USGIN users are encouraged to encode their data using GeoSciML-portrayal where possible; GeoSciML-portrayal is a simplified, distilled version of GeoSciML. For more information, see the GeoSciML-Portrayal Cookbook.

GeoSciML-Portrayal [Dead Link, https://www.seegrid.csiro.au/wiki/CGIModel/GeoSciMLPortrayalViewModel]

A simplified, distilled version of GeoSciML; includes schemas and vocabularies for simple features. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/gsmlp] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

GeoSciML-portrayal is ideal as an interchange format for USGIN users to encode their data, either for exchange between applications or for deployment as a web service. For more information, see the GeoSciML-Portrayal Cookbook.

IETF URI

A Uniform Resource Identifier (URI) is a compact sequence of characters that uniquely identifies an abstract or physical resource; URI standards were developed by the Internet Engineering Task Force (IETF). For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/ietf-rfc-3986-universal-resource-identifier] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

USGIN users usually employ URIs when mapping their data to a USGIN schema. In this context, generic IETF URIs are less optimal than USGIN URIs; USGIN URIs conform to a USGIN-specific profile of the IETF URI standard.

W3C Extensible Markup Language (XML)

A versatile markup language developed by the World Wide Web Consortium. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/w3c-extensible-markup-language-xml] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

XML is the basis for the specialized markup languages, interchange formats, and schemas used by USGIN; these include GML, GeoSciML, and GeoSciML-portrayal. Any time these specialized markup languages are used, XML is used. For more information about XML, see the USGIN XML Tutorial
Category: Web Service Profiles
Specification Description Uses

GeoSciML WFS Architecture [Dead Link, http://lab.usgin.org/profiles/geosciml-wfs-architecture]

Profile for interoperable Open Geospatial Consortium (OGC) web feature services based on the GeoSciML schema; USGIN users who wish to structure their data using GeoSciML and deploy this data as an OGC web feature services (WFS) should accommodate this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/geosciml-wfs-architecture] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

GeoSciML-portrayal Simple Feature Schemas are schemas developed by USGIN used to structure data describing geophysical features. USGIN users will use the schemas indicated by this profile to structure their data for maximum interoperability with other USGIN web services. For more information, see the GeoSciML-Portrayal Cookbook.

OGC CSW APISO - OGC ISO19115/ISO19119 application profile for CSW 2.0

A profile of the OGC CSW 2.0.2 standard. Users who wish to deploy an OGC-compliant catalog service for their data should do so in accordance with this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/ogc-csw-apiso-ogc-iso19115iso19119-application-profile-csw-20] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Users typically deploy catalog services to list and manage queries regarding a large number of different web-accessible resources. Compliance with this profile of OGC specifications maximizes compatibility with other OGC services and ensures a degree of compatibility with USGIN services.

Profile for basic wells and borehole information service [Dead Link, http://lab.usgin.org/profiles/profile-basic-wells-and-borehole-information-service]

Content model and specifications for data describing boreholes; uses GeoSciML Borehole encodings. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/profile-basic-wells-and-borehole-information-service] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

AASG Geothermal Data [Dead Link, http://www.stategeothermaldata.org/] subrecipients who wish to share data describing wellbores or boreholes do so in accordance with the content model and specifications indicated by this profile

USGIN CSW Profile [Dead Link, http://lab.usgin.org/profiles/usgin-csw-profile-home]

Profile detailing a web service interface to search metadata catalogs; USGIN users who wish to deploy a USGIN-compliant catalog service for their data should accommodate this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/usgin-csw-profile-home] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

Users typically deploy catalog services to list and manage queries regarding a large number of different web-accessible resources. Compliance with this profile when deploying a catalog service ensures maximum compatibility with other USGIN services

USGIN Vocabulary Service Profile [Dead Link, http://lab.usgin.org/profiles/usginvocabularyservice]

Profile for USGIN-compliant vocabulary service. Users who wish to deploy a USGIN-compatible vocabulary service should accommodate this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/usginvocabularyservice] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

USGIN users deploying datasets containing specialized or personal vocabulary terms should define those terms in a web-accessible way, so other users accessing these datasets have the opportunity to look up vocabulary terms with which they are unfamiliar. This profile provides an optimal, USGIN-compatible method for deploying a vocabulary service that performs these functions.

USGIN Web Map Service Profile [Dead Link, http://lab.usgin.org/profiles/usgin-wms-profile]

Profile for interoperable web map services; USGIN users who wish to deploy their data as USGIN-compliant web map services should accommodate this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/usgin-wms-profile] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

As web map services are typically used to deploy raster datasets, this profile will be of use to any USGIN user wishing to share data that can be expressed as a raster image; using this profile will maximize WMS compatibility with other USGIN services
Category: Metadata Profiles
Specification Description Uses

USGIN ISO Metadata Profile [Dead Link, http://lab.usgin.org/profiles/usgin-iso-metadata-profile]

USGIN profile for the ISO19139 metadata standard. USGIN users with data to share who wish to describe their live data using metadata records in the USGIN Catalog [Dead Link, http://catalog.usgin.org/search/] or an equivalent Catalog Service for the Web should conform metadata records describing their data to this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/usgin-iso-metadata-profile] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

USGIN technical users will use metadata records to describe and register their data resources in the USGIN Catalog; these metadata records should conform to the USGIN profile of the ISO19139 metadata standard. For more information, see the USGIN Subdomains entry in the USGIN FAQ.
Category: URI Profiles
Specification Description Uses

USGIN Uniform Resource Identifier Profile [Dead Link, http://lab.usgin.org/profiles/usgin-uniform-resource-identifier-profile]

Recommendations for usage and coining of Uniform Resource Identifiers (URIs). Any USGIN user creating URIs should conform their URIs to this profile. For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/profiles/usgin-uniform-resource-identifier]on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.

USGIN users create URIs when mapping their data to a USGIN schema. USGIN URIs are used to identify certain resources and dereference to representations of these resources.

Specification Description Uses

W3C Simple Knowledge Organization System (SKOS)

A standard way to represent knowledge organization systems using the Resource Description Framework (RDF). For more information, see the corresponding entry [Dead Link, http://lab.usgin.org/standards/w3c-skos] on the USGIN Laboratory [Dead Link, http://lab.usgin.org/] website.
