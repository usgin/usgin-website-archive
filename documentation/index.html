USGIN Specification Documentation

This page provides a list of documents describing USGIN specifications, schemas, and model artifacts. Software development projects are all publicly accessible on the USGIN GitHub. The current process for adopting specifications is quite informal because the user community is small. The introduction of specifications that duplicate the capabilities of existing specifications is discouraged, unless doing so provides for the utilization of newer technology.

See also the Lab USGIN Content Archive for additional specifications, tutorials, presentations, and scripts recovered from the lab.usgin.org Drupal site.

Category: Metadata and Catalog
Specification Description

Metadata Recommendations for Geoscience Resources

Provides guidance on the metadata content required to meet the use requirements for USGIN metadata; not specific to any particular metadata standard or implementation.

Use of ISO metadata specifications to describe geoscience information resources

Describes recommended practices for using ISO19139 XML encoding of ISO 19115 and ISO 19119 metadata to describe a broad spectrum of geoscience resources.

USGIN metadata best practices [Dead Link, http://repository.usgin.org/uri_gin/usgin/dlio/499]

Guidelines for implementation of USGIN metadata content recommendations in ISO 19139 XML metadata.
Category: Information Exchange
Specification Description

Simple Feature Content Model Guidelines [Dead Link, http://repository.usgin.org/uri_gin/usgin/dlio/340]

Guidance for creating an Excel workbook that defines a content model for a flat file data interchange format, to be implemented in XML as a GML simple feature.

NGDS content models [Dead Link, http://schemas.usgin.org/home/]

A directory of the information exchange content models in use by the National Geothermal Data System [Dead Link, http://geothermaldata.org/].
Category: System
Specification Description

URI policies

Presents a proposal for minting of dereferenceable HTTP URIs for use in interoperable web services.
