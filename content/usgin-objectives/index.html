USGIN Objectives

The objectives of USGIN are as follows:

  1. To facilitate public access to interoperable digital geoscience data
  2. To reduce the cost of online data publication
  3. To preserve ownership, credit, and control of existing data
  4. To distribute the logistical overhead associated with sharing digital information
  5. To minimize reliance on proprietary software applications

These objectives are explained in detail below.

In order to accomplish these objectives, USGIN participants voluntarily comply with USGIN specifications.

Public access to interoperable digital geoscience data

Broken down, “public access to interoperable digital geoscience data” connotes the following:

  1. Geoscience data is data that describes geophysical resources like petroleum, natural gas, or geothermal energy
  2. Digital data is data that is stored in a computer
  3. Interoperable data can be processed by a computer with minimal user interpretation or input
  4. Public access to data is achieved when the general public can access the data with little more than a web browser

This objective can be divided into three tiers:

  • Tier 1: Data is web-accessible and discoverable; this tier can be divided into three parts:
    • Tier 1.1: Data Digitization
    • Tier 1.2: Web Accessibility
    • Tier 1.3: Discoverability
  • Tier 2: Data discovery is interoperable
  • Tier 3: Data analysis is interoperable

Tier 1: Web-Accessibility and Discoverability

Data is web-accessible when a user with an Internet connection can access the data; data is discoverable when users can find it.

Tier 1.1: Digitization

In order to be web-accessible, data must first be digitized – that is, converted from a physical object such as a map, photograph, or well log, into a computer file of some kind. This can be both labor- and resource-intensive, particularly in situations in which analog data has not been adequately inventoried.

The digitization process usually involves a combination of scanning analog documents and manually entering data into appropriate software applications. For example:

  • Analog maps can be scanned as raster images; some maps require a wide-bed scanner
  • Maps scanned as raster images can be georeferenced using GIS software
  • Georeferenced digital maps can be manually vectorized as points, lines or polygons by human operators using GIS software
  • Well logs can be digitized via a combination of scanning and hand-digitization
  • Data records that use quantitative or qualitative values to describe geophysical resources can be manually entered into database applications

In general, agencies participating in the USGIN initiative use their own resources and staff to digitize any data they wish to share.

Tier 1.2: Web Accessibility

Data must be physically stored on a server in order to be web-accessible. A server is a computing platform that is configured to listen for requests over the Internet and respond to those requests by serving the requested data.

USGIN was designed to facilitate web-accessibility for a very large amount of data nationwide; to accomplish this, large amounts of data must therefore be moved to a large number of servers. This begs the following questions:

  • How will these servers be provided and maintained?
  • Where will these servers be physically located?
  • What will the capabilities of these servers be, and how will they be updated over time?
  • How will users upload their data to these servers?
  • How will an inventory be kept of the contents of these servers?
  • Who owns the data once it has been uploaded to a given server?

To circumvent these problems, USGIN specifies that any server may be used to host USGIN data, provided the data and the server are appropriately configured. This allows users to host and control their own data, or to host their data using existing server resources (such as servers provided by Amazon Web Services).

Tier 1.3: Discoverability

Users must also be able to discover web-accessible data.

Data discovery is facilitated by a user interface that is capable of providing users with lists of data resources. In general, the best way to facilitate data discovery is via a catalog of some kind: web-accessible data should be listed in a catalog to help users discover available data.

The result: a distributed network of discoverable, web-accessible data.

Tier 2: Interoperable Data Discovery

Beyond data accessibility and discoverability, USGIN facilitates interoperable data discovery -- that is, data discovery that does not require the user to demonstrate significant knowledge of USGIN architecture.

Interoperable data discovery generally requires the following:

  • Data entered in tabular format (that is, in a table: an Excel spreadsheet, an HTML or XML table, or a database)
  • Data available via Open Geospatial Consortium (OGC) web services
  • Data described by metadata records registered in a web-accessible catalog, often a catalog service for the web
  • Data and metadata are structured in such a way that they can be ingested and processed by software applications with a minimum of user knowledge and input

USGIN specifies Open Geospatial Consortium (OGC) web services as a primary mechanism for transferring web-accessible data. USGIN web services must also be described by metadata records registered in a web-accessible catalog such as the USGIN Catalog [Dead Link, http://catalog.usgin.org/search] (Figure 1). Registered metadata records list the properties of the data resources they describe, allowing users to find data based on those properties.

Tier 3: Interoperable Analysis

In general, users search for and access data in order to perform analyses using the data; within the context of geoscience data, these are typically geospatial analyses.

The process of analysis becomes complicated when different datasets descibing the same concept are structured according to different schemas. Complications include the following:

  • Identical concepts described using different vocabulary terms (example: pyrite is used in one dataset, fool's gold in another; both terms are used to identify an iron sulfide)
  • Disparity in units of measure (example: meters are used in one dataset, feet are used in another)
  • Descrepancies in field headers and content (example: Longitude is used as a field header in one dataset, Long. is used as a field header in another; both fields contain longitudinal measurements)
  • Lack of globally unique identifiers to differentiate between features and vocabulary terms and facilitate comparisons between databases

Machines are incapable of compensating for discrepancies like these; user interpretation and mediation are necessary before analysis can begin. For a more thorough treatment on this subject, see the AASG Geothermal Data Content Model Tutorial.

USGIN circumvents the above issues by defining interchange formats, usually XML documents and schemas, to structure USGIN data. In other words, data that conforms to USGIN specifications will be structured according to a predetermined schema that is designed to facilitate maximum interoperability.

Review

USGIN data is:

  • Web-accessible
  • Discoverable
  • Tabular
  • Available via OGC web services
  • Described by metadata records registered in a web-accessible catalog
  • Interoperable
  • Structured as an interchange format

As a result of these characteristics, USGIN data is capable of interoperable discovery and analysis.
