What can USGIN do for me?

USGIN provides the following:

  • Open-source access to geoscience data from across the country
  • Specifications and best practices for deploying web-accessible, interoperable data resources as part of the network

USGIN data resources can be accessed with a standard web browser, without proprietary software, and USGIN supports open-source solutions for data deployment.

Consequently: if you wish to access USGIN data resources, you can do so with little more than web browser. If you are a technical user with data resources you wish to deploy as part of USGIN, you can do so with open-source software; tutorials are available on this web site.

Who uses USGIN?

USGIN users can be divided into two general categories:

  • Basic users, who want to discover, access, and explore USGIN data; anyone can be a basic user, from a student to a casual browser to an industry professional
  • Technical users, including data providers and data developers

USGIN may be used by anyone with an Internet connection.

For more information, see Using USGIN.

By far the largest implementation of USGIN specifications is the National Geothermal Data System (NGDS) [Dead Link, http://www.geothermaldata.org], a Department of Energy (DOE)-funded effort to digitize and share geoscience data from around the country. NGDS is built according to USGIN specifications and therefore part of USGIN.

When fully operational, NGDS will contain data describing:

  • 2,000,000 wells
  • 1,000,000 feet of core samples
  • 500,000 Bottom Hole Temperature (BHT) measurements
  • 100,000 well logs
  • 50,000 geothermal wells
  • 10,000+ scanned publications and maps

All of this data will be live and accessible to anyone, using nothing more than a web browser and open-source software. Digitization is ongoing and tracked by a live progress report [Dead Link, http://services.usgin.org/track/map].

How do I use USGIN?

Basic users can discover and access USGIN data resources just by browsing the USGIN Catalog [Dead Link, http://catalog.usgin.or/search] (for more information about the USGIN Catalog, see below).

Technical users who wish to share their data as part of USGIN can use the resources on this website to deploy their data in compliance with USGIN specifications. In addition to USGIN specifications, this website also provides tutorials, best practices, and a workflow outline for prospective data providers.

For more information, see Using USGIN.

How does USGIN work?

USGIN can be conceived in three parts:

  • USGIN-compliant data resources
  • USGIN subdomains, partners, and affiliates
  • USGIN specifications

USGIN is comprised by data resources shared in accordance with USGIN specifications. The more data shared this way, the larger the network grows.

USGIN users access and share data resources via the utilities available on USGIN subdomains and affiliated web sites. These utilities include the USGIN Repository, the USGIN Catalog, and the USGIN Lab. For more information on the USGIN subdomains, see below.

What are USGIN Specifications?

USGIN specifications are the standards, practices, and protocols employed by USGIN to address the challenges associated with sharing large amounts of diverse data stored in geographically disparate locations. Compliance with USGIN specifications allows technical users to share their data in such a way that the data they share is interoperable.

USGIN specifications include:

  • Catalog systems for data discovery on a distributed network
  • Recommended data file formats that can be accessed by multiple different client applications
  • Networking protocols used to make data available on the Internet and accessible to various web browsers and client applications
  • Web service definitions that define interfaces for searching catalogs and accessing data resources
  • Client software applications that are recommended or supported by USGIN
  • International standards used for USGIN data development
  • Schemas used for databases and interchange files to encourage interoperability at the data level

For a more specific list of USGIN specifications, see the USGIN Specifications page.

What are USGIN Subdomains?

USGIN Subdomains are websites within the USGIN domain (USGIN.org) that provide USGIN users with utilities. There are three USGIN subdomains of note:

  • The USGIN Catalog (http://catalog.usgin.org/search/ [Dead Link, http://catalog.usgin.org/search/]): USGIN is a distributed network, so data resources that are part of the network are not stored in a central location. Consequently, the only way to know what is on the network is via a list of registered USGIN data resources. The USGIN Catalog provides this list by allowing USGIN users to manage and search metadata records describing registered USGIN data resources. If a data resource is furnished with a metadata record in the USGIN Catalog and shared according to USGIN specifications, this data resource is then part of the network and can be found in the USGIN Catalog.
  • The USGIN Lab (http://lab.usgin.org/ [Dead Link, http://lab.usgin.org/]): A website that acts as an informal USGIN workshop and provides users with bulletin boards to discuss USGIN-related issues and interact with USGIN developers
  • The USGIN Repository (http://repository.usgin.org/ [Dead Link, http://repository.usgin.org/]): A website used by USGIN users to manage and store static or legacy documents that cannot be deployed as USGIN-compliant web services; the USGIN Repository is also used to enter and manage metadata records describing USGIN-compliant data resources before these metadata records are harvested by the USGIN Catalog

Why USGIN?

Sharing large amounts of digital data from multiple separate sources poses a number of challenges. For example:

  • Data resources that are stored in geographically distributed locations must become publicly accessible without logistical complications
  • The public must be able to access shared data resources without the need for proprietary software applications
  • Users must be able to perform a search for desired data resources; search terms must be applicable to all shared data resources

USGIN specifications address these challenges. For more information, see the USGIN Specifications page.
