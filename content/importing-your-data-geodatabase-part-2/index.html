Importing your Data into a Geodatabase, Part 2

Importing data into the geodatabase from an Excel file via Microsoft Access

To import your data into the newly created geodatabase from an Excel file, take the following steps:

  1. Close ArcCatalog
  2. Open the personal geodatabase you just created in Microsoft Access
  3. Click the External Data tab at the top of the Microsoft Access window. (Figure 4)
  4. Click the Import Excel Spreadsheet button (Figure 4)

    Figure 4: The External Data tab in Microsoft Access

  5. In the Get External Data window that appears (Figure 5), click Browse. Look through your computer and find your data. Your data should be in an Excel workbook (.xls) file. When you have found your data, click Open. Make sure the Import the source data into a new table in the current database radio button is selected. Then click OK in the Get External Data window (Figure 5). This will bring up the Import Spreadsheet Wizard

    Figure 5: The Get External Data window in Microsoft Access

  6. In the Import Spreadsheet Wizard window that appears (Figure 6), use the box at the top of the window to select the spreadsheet that corresponds with your data. In the example in Figure 6, the appropriate spreadsheet is BoreholeObservation, but the appropriate spreadsheet will be different depending on the content model used

    Figure 6: Choosing your data source

  7. In the next window of the Wizard, click the First Row Contains Column Headings checkbox (Figure 7). When this box is checked, click Next

    Figure 7: This box should be checked before you click Next

  8. The next window (Figure 8) allows you to format specific fields. Do nothing. Click Next

    Figure 8: Do nothing except click Next

  9. The next window (Figure 9) is rather more important. In this window, Microsoft Access wants to know if it should assign a primary key to each record. A primary key is a number used by Microsoft Access to organize its database records, ensure that each record is unique, and allow other databases to refer to specific records within a database. Under some circumstances, you would want Access to do this for you, but not this time: because of the schemas used by State Geothermal Data (SGD) [Dead Link, http://www.stategeothermaldata.org]content models, your data won’t need a primary key. Click No primary key (Figure 9) and then click Next

    Figure 9: Click the No Primary Key radio button

  10. Type a name for your table (Figure 10), or use the default one. Then click Finish.

    Figure 10: Finish the importing process
